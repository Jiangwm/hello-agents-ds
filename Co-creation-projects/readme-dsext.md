# Hello-Agents 共创项目的工业数据分析与挖掘迁移建议

## 1. 分析范围与方法

本文对 `Co-creation-projects/` 下 40 个实际应用项目进行源代码静态分析，主要查看各项目的 README、Notebook、Agent/Tool、数据处理、接口与前端文件；未安装依赖、未配置外部 API、未运行模型或服务。因此，下文描述的是代码所体现的设计能力，不代表生产环境可用性。`EXAMPLE-ProjectTemplate` 仅为贡献模板，不计入应用项目。

迁移目标统一理解为：面向制造企业的设备、工艺、质量、能耗、生产、供应链和运维数据，形成“数据接入—治理—分析/挖掘—解释—决策—反馈”的闭环。涉及生产控制的方案应默认采用只读接入、权限隔离、结果可追溯和人工确认，不允许大模型直接下发高风险控制指令。

## 2. 项目逐条分析

### 1. `1zrj-DataAnalysisAgent`

- **项目简介**：基于 HelloAgents `SimpleAgent` 的通用表格分析助手。`main.ipynb` 负责读取数据、生成统计分析与 ECharts 可视化代码，并输出 Markdown 报告和 HTML 图表。
- **迁移建议**：替换示例 Excel 为设备时序、工序质量、能耗或产量数据；补充时间戳对齐、缺失值/漂移检测、控制图、相关性与异常点工具。报告模板固定为 OEE、良率、停机、单位能耗、异常清单和处置建议，并保留每项结论对应的数据区间与计算公式。

### 2. `939147533-DatabaseAgent`

- **项目简介**：自然语言转 SQL 的 Oracle 查询助手。`src/react_agent.py` 组织 ReAct 流程，`src/tools.py` 提供数据库结构读取、SQL 生成、查询执行与结果格式化。
- **迁移建议**：接入 MES、ERP、QMS、WMS 或设备历史库，建立统一的工业指标语义层，让“某产线上周良率下降原因”等问题可被稳定映射到表、字段和口径。生产化前必须限制为只读账号、SQL 白名单、行数/耗时上限、敏感字段脱敏，并在执行前显示 SQL 与数据口径供人工确认。

### 3. `afei-GuessWhoAmI`

- **项目简介**：FastAPI + Web 前端的人物竞猜应用，包含人物生成、角色扮演、多轮会话、提示生成、语义判定和会话状态管理；搜索工具用于补充人物资料与图片。
- **迁移建议**：将“猜人物”改造成“猜故障/辨工况”训练系统：系统从历史故障库抽取一个案例，学员通过询问振动、温度、报警、维修史等特征定位故障。原提示机制可改为分层证据披露，语义判定改为故障编码/同义词匹配，最终输出诊断路径、遗漏证据和标准处置流程，用于维修人员培训。

### 4. `alexrunner-DataAnalysisAgent`

- **项目简介**：商品销售数据分析智能体。`main.py` 串联规划、ReAct 分析和报告 Agent，`tools/data_exploration.py`、`tools/data_analysis.py` 提供数据探查、分组统计、趋势分析和图表能力。
- **迁移建议**：保留“规划—执行—汇总”架构，将商品维度替换为工厂、产线、设备、产品、班次和批次，将消费指标替换为产量、节拍、良率、停机、返工与能耗。新增时序切分、设备层级聚合、SPC、Pareto、异常检测和根因候选排序工具，避免 LLM 直接编造统计结论。

### 5. `allen2000-FashionDailyDress`

- **迁移建议**：改造成“环境—生产策略建议系统”：天气 Agent 替换为厂区温湿度、气象、峰谷电价等数据工具；穿衣 Agent 替换为工艺/排产建议 Agent；协调器综合设备能力、订单优先级和环境约束，给出烘干、涂装、冷链或能源调度建议。输出应附约束条件、置信度和人工审批入口。

## 4. 共性落地路线

1. **先统一数据语义**：建立工厂—产线—设备—部件—测点—产品—批次层级，以及 OEE、良率、停机、能耗等指标口径。
2. **把计算交给确定性工具**：SQL、统计、异常检测、预测和绘图由可测试的 Python/数据库工具执行，LLM 负责规划、解释和报告组织。
3. **保留完整证据链**：每条结论记录数据源、查询/代码、时间范围、样本量、模型版本、阈值和引用文档。
4. **建立工业验证集**：使用已脱敏且有专家结论的历史案例，分别评估数值正确性、根因召回、误报/漏报、报告可读性和处置可执行性。
5. **分阶段上线**：先离线分析与报告，再只读在线辅助，最后才考虑经审批的工单联动；PLC、DCS、机器人等控制系统始终设置人工确认和独立安全联锁。
6. **落实治理要求**：执行最小权限、数据脱敏、网络分区、提示/模型版本管理、审计日志、失败降级和人工复核，防止敏感数据外泄及错误建议进入生产环节。
