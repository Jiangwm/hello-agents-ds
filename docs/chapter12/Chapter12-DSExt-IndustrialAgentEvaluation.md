# 第十二章 DSExt：工业数据分析智能体评测体系

> 难度：高阶 ｜ 对应主题：BFCL、GAIA、数据生成、LLM Judge、Win Rate 与人工验证

## 1. 场景定位

为前述工业智能体建立一套离线评测：既验证工具调用的结构正确性，也验证多步骤分析、证据引用、报告质量、安全性和成本。评测对象可以是不同模型、提示词、工具版本或 Agent 架构。

本场景强调“分层评测”，避免用一个总分掩盖高风险失败。

## 2. 与原章内容的静态映射

| 原章知识或代码证据 | 工业扩展映射 |
| --- | --- |
| BFCL 工具调用评估脚本 | 工业工具选择、参数、顺序和多轮调用评估 |
| GAIA 多步骤任务 | 跨数据表、规范和日志的综合分析任务 |
| `AIMEGenerator` | 受模板和约束控制的工业案例生成器 |
| LLM Judge 与 Win Rate 报告 | 评估报告质量并做版本成对比较 |
| `HumanVerificationUI` | 让领域专家复核证据、风险和可执行性 |
| 综合报告生成流程 | 汇总分层指标、失败类型和改进建议 |

## 3. 评测任务分层

### L1：单工具调用

验证工具名、必填参数、类型、单位、时间窗和错误处理，类似工业版 BFCL。

### L2：多工具链路

要求智能体先检查数据质量，再计算趋势并查询规范；评估顺序、冗余调用和中间状态。

### L3：开放分析任务

给出一组数据与业务问题，要求界定异常、提出假设、寻找反证并生成报告，类似面向工业问题的综合基准。

### L4：安全与拒答

构造越权控制、敏感数据、证据缺失、单位冲突和提示注入案例，验证安全边界。

## 4. 指标体系

- 工具：名称准确率、参数完全匹配率、有效调用率；
- 分析：关键步骤覆盖率、证据召回率、假设可反证性；
- 报告：事实正确率、引用正确率、结论校准度；
- 安全：越权率、无依据建议率、敏感信息泄漏率；
- 效率：调用次数、令牌、延迟和估算成本；
- 稳定性：重复运行一致性、格式敏感性和数据扰动敏感性。

高风险安全指标采用门槛制，不与其他得分简单平均。

## 5. 数据生成与人工验证

生成器只能基于批准的模板、故障机制和参数范围创建候选题；随后进行：

1. 结构与数值一致性检查；
2. 去重与训练集相似度检查；
3. LLM Judge 初筛并保留评判理由；
4. 专家在验证界面中批准、修改或拒绝；
5. 固化数据版本和答案版本。

LLM Judge 不能独立裁决安全性和领域正确性。

## 6. 对比实验

对两个智能体版本进行盲化成对比较，随机答案顺序，统计胜、平、负和置信区间。除整体胜率外，按任务层级、设备类型、风险等级和失败类别分层报告。

## 8. 验收标准

- 至少覆盖单工具、多工具、开放分析和安全拒答四层任务；
- 所有测试样例有版本、来源与专家审核状态；
- 报告同时给出分数、置信区间和失败案例；
- 安全门槛不因综合均分较高而被忽略；
- LLM Judge 结果可被人工复核和覆盖。
