# 第十三章 DSExt：工厂能源分析与排产建议助手

> 难度：综合 ｜ 对应主题：数据模型、多智能体、MCP、FastAPI、前后端分离与交互功能

## 1. 场景定位

将“旅行规划”的约束规划思路迁移到工厂能源场景：用户选择产线、日期、产量目标、峰谷电价和不可调整工序，系统汇总能耗数据，预测不同排产方案的成本与峰值，并生成可编辑的建议方案。

系统提供决策支持，不直接下发排产或控制设备；所有建议必须显示数据范围、约束和估算误差。

## 3. 数据模型

核心输入模型：

- `PlanningRequest`：产线、计划日期、产量目标、班次和优化目标；
- `OperationConstraint`：工序顺序、持续时间、可移动窗口和资源约束；
- `TariffPeriod`：峰平谷时段与电价；
- `EnergyBaseline`：设备或工序的功率分布和置信区间；
- `ProductionTask`：产品、批量、设备候选和交付时间。

核心输出模型：`ScheduleOption`、`EnergyCurve`、`CostBreakdown`、`ConstraintViolation`、`RecommendationEvidence`。

## 4. 多智能体职责

- 数据智能体：检查计量点完整性、时间对齐和异常值；
- 预测智能体：估算任务功率曲线及不确定性；
- 优化智能体：在业务约束下生成多个候选排产；
- 审核智能体：复算成本、检查约束和比较基线；
- 报告智能体：将结构化结果转为面向生产人员的说明。

确定性约束检查和成本计算由程序完成，大模型负责解释、补充问题和组织报告。

## 5. 应用流程

1. 用户提交计划与约束；
2. 后端验证数据模型和配置；
3. MCP 客户端读取历史能耗、生产计划和电价；
4. 数据智能体形成可用数据摘要；
5. 预测与优化模块生成基线和候选方案；
6. 审核智能体检查约束、计算和证据；
7. 前端展示成本、峰值、单位能耗与时间轴；
8. 用户编辑锁定任务后重新计算并导出。

## 6. 前端与交互

- 首页：产线、日期、产量目标、优化目标和约束表单；
- 进度：显示取数、校验、预测、优化和审核状态；
- 结果：基线与候选方案的能耗曲线和指标卡；
- 编辑：锁定或移动允许调整的任务并触发复算；
- 导出：输出方案、数据版本、模型版本、约束和风险。

## 8. 验收标准

- 请求和响应由明确的数据模型约束；
- 任一方案均通过确定性约束检查和成本复算；
- 页面能追踪数据版本、假设与估算误差；
- 用户编辑后只重算受影响部分并保留变更记录；
- 系统不直接连接生产排程或设备控制写接口。
