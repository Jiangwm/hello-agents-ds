# 第十一章 DSExt：工业诊断智能体的 SFT 与 GRPO 优化

> 难度：高阶 ｜ 对应主题：数据集、奖励函数、LoRA、SFT、GRPO、评估与部署

## 1. 场景定位

已有工业诊断智能体能调用数据工具，但经常出现工具选择错误、参数范围不合理、报告格式不稳定等问题。本场景构造脱敏训练任务，先用 SFT 学习标准调查轨迹，再用 GRPO 优化工具调用正确性、诊断证据性和响应成本。

训练目标是改进“分析行为”，不是让模型直接学习真实设备控制策略。数据和奖励都必须经过领域专家审查。

## 3. 训练任务与数据格式

每个样例包含：

- 工业问题、设备范围和允许工具；
- 必需证据与禁止推断项；
- 参考工具调用序列及参数；
- 工具观察结果；
- 参考报告和可接受的等价答案；
- 难度、风险等级、数据来源和专家审核状态。

数据划分应按设备类型、故障族或时间切分，防止相似工单泄漏。高风险案例单独建立保留集。

## 4. 奖励函数设计

总奖励由多个可解释分量组成：

- **工具选择奖励**：选择了完成任务所需的工具；
- **参数奖励**：设备、时间窗、字段和单位正确；
- **证据奖励**：结论引用了正确观察结果；
- **诊断奖励**：候选原因排序符合专家标签或允许集合；
- **校准奖励**：证据不足时降低置信度或拒答；
- **格式奖励**：输出满足结构化模式；
- **效率奖励**：惩罚重复、无效或超预算调用；
- **安全惩罚**：越权控制、伪造数据或忽略高风险提示时给予强惩罚。

避免只按最终故障标签奖励，否则模型可能学会猜答案而不形成正确调查过程。

## 5. 实验流程

1. 建立不训练的基础模型基线；
2. 用高质量专家轨迹进行 SFT；
3. 检查 SFT 是否改善格式但损害探索能力；
4. 在隔离环境中运行 GRPO，监控各奖励分量、KL 和输出长度；
5. 对基础、SFT、GRPO 三个版本做盲测与错误分析；
6. 只有通过高风险保留集和人工审查后，才进入离线灰度验证。

## 6. 风险与治理

- 奖励黑客：格式正确但证据错误；
- 数据污染：相似工单跨训练与测试集；
- 灾难性偏移：常见故障提升但罕见高风险故障退化；
- 专家分歧：参考答案并非唯一，需要允许答案集合；
- 生产边界：模型只给建议，不直连设备执行器。

## 8. 验收标准

- 数据集说明来源、划分策略和专家审核状态；
- 奖励能分解到工具、参数、证据、格式、成本和安全；
- 对比实验报告包含均值、分布和失败案例；
- 高风险能力没有因平均分提升而被掩盖；
- 不将训练后的模型直接部署到生产控制环节。
