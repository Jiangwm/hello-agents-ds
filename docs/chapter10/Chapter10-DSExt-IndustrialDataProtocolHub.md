# 第十章 DSExt：工业数据协议协作中枢

> 难度：进阶 ｜ 对应主题：MCP、A2A、ANP、自定义服务器与协议选型

## 1. 场景定位

工业数据分散在时序数据库、MES、QMS、维修系统和文档库中。本场景构建一个教学型协议中枢：通过 MCP 暴露标准化只读数据工具，通过 A2A 让专业分析智能体协作，通过 ANP 演示服务发现与任务分配。

协议层负责能力描述、身份、传输和调用，不掩盖源系统的数据语义、权限和质量问题。

## 3. MCP 工具设计

首个服务器只提供脱敏样例数据，工具包括：

- `list_equipment()`：列出设备及可用测点；
- `query_timeseries()`：按设备、测点和时间窗聚合查询；
- `get_batch_quality()`：返回批次质量摘要；
- `lookup_alarm_event()`：按告警码和时间范围查询事件；
- `get_data_dictionary()`：返回字段、单位和采样频率。

每个工具定义类型、单位、最大时间窗、最大行数和错误语义。默认不设计 `set_parameter`、`start_machine` 等控制工具。

## 4. A2A 协作网络

- 数据质量智能体：检查完整性、时间对齐和单位；
- 时序分析智能体：发现漂移、突变和周期异常；
- 设备诊断智能体：结合告警与维修知识形成假设；
- 报告智能体：合并证据并生成可审计结论。

A2A 消息应携带任务 ID、能力声明、输入数据范围、证据引用、状态和错误。任务转交不能只发送自然语言摘要，否则下游无法验证数据范围。

## 5. ANP 服务发现练习

模拟两台具有相同能力但数据域和负载不同的分析服务。发现和选择策略考虑：

- 是否覆盖目标产线与时间范围；
- 数据权限与租户隔离；
- 支持的算法版本；
- 当前负载、超时和健康状态；
- 结果可追溯性。

## 6. 安全与可观测性

- 身份认证、最小权限和工具级授权；
- 输入模式校验、查询窗口限制和速率限制；
- 请求 ID 贯穿 MCP、A2A 和 ANP；
- 记录服务、工具、参数摘要、响应大小和错误；
- 敏感字段在协议边界前脱敏；
- 真实控制能力必须置于独立安全域并经过人工审批。

## 8. 验收标准

- 能解释同一场景中 MCP、A2A、ANP 的不同职责；
- 工具模式包含单位、范围和明确错误；
- 跨协议调用可用同一请求 ID 追踪；
- 所有教学工具默认只读且基于脱敏样例；
- 服务不可用时有超时、重试边界和降级说明。
