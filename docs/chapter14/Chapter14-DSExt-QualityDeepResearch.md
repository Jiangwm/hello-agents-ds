# 第十四章 DSExt：质量异常自动化深度研究智能体

> 难度：综合 ｜ 对应主题：TODO 驱动研究、专业 Agent、搜索、笔记、服务层与实时进度

## 1. 场景定位

当某类缺陷突然升高时，工程师需要综合内部批次数据、设备事件、工艺规范、历史 8D 报告和供应商资料。本场景构建一个 TODO 驱动的深度研究智能体，围绕“缺陷从何时开始、关联什么变化、有哪些可反证假设”生成证据化调查报告。

研究范围优先使用本地脱敏资料；若允许外部检索，外部来源只能补充通用机理，不能替代内部批次证据。

## 2. 与原章内容的静态映射

| 原章知识或代码证据 | 工业扩展映射 |
| --- | --- |
| `TodoItem` 与 TODO 三阶段流程 | 将质量问题拆成可追踪的调查任务 |
| `DeepResearchAgent` | 协调规划、取证、笔记、总结和报告 |
| `PlanningService` | 生成数据、工艺、设备、物料和文档调查线 |
| `dispatch_search()` | 调度内部数据检索和允许的外部搜索 |
| `build_note_guidance()` | 统一证据笔记格式和引用要求 |
| `SummarizationService`、`ReportingService` | 分阶段压缩并生成最终 8D 风格草稿 |
| `ToolCallTracker` 与前端进度 | 展示实时任务、工具和失败状态 |

## 3. TODO 研究模型

每个任务包含：

- 问题、数据范围、所需工具和依赖任务；
- 状态：待处理、执行中、阻塞、待人工确认、完成；
- 证据引用、负面结果和来源质量；
- 候选结论、置信度与反证条件；
- 时间、调用成本和失败原因。

典型任务线：异常范围界定、正常对照构造、工艺参数漂移、设备与维护事件、物料批次变化、历史相似案例、候选根因验证。

## 4. 三阶段流程

### 规划

把用户问题转成具备依赖关系的 TODO；人工确认调查范围、数据权限和最大预算。

### 研究

按 TODO 调用只读工具，保存事实、表格摘要和来源。搜索无结果也是有效证据，必须记录查询条件。发现关键数据缺失时转为阻塞，不凭常识填空。

### 报告

按“问题定义—影响范围—数据证据—候选根因—反证—结论—验证计划”生成报告。报告服务只使用已批准笔记，不能绕过证据库重新自由生成。

## 5. 工具与来源

- 批次和质量数据库查询；
- 工艺时序分析；
- 设备告警与维护事件检索；
- 本地工艺规范、历史 8D 和供应商文档检索；
- 可选的外部论文或公开技术资料检索；
- 表格、图表和证据包导出。

所有来源记录 URI 或路径、版本、检索时间、数据范围和权限级别。

## 6. 实时交互与人工关口

前端展示 TODO 树、当前工具、已用预算、证据数量和阻塞原因。以下节点必须人工确认：研究计划、跨域数据访问、追加预算、验证实验和最终根因结论。

## 8. 验收标准

- TODO 具备状态、依赖、证据和失败原因；
- 报告中的关键陈述可追溯到批准笔记；
- 工具无结果、数据冲突和预算耗尽均有显式路径；
- 人工可暂停、修改范围并恢复任务；
- 内部事实与外部通用知识在报告中清晰分区。
