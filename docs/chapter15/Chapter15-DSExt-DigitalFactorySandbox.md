# 第十五章 DSExt：多智能体数字工厂分析沙箱

> 难度：综合高阶 ｜ 对应主题：多角色智能体、记忆、关系、状态管理、FastAPI 与可视化前端

## 1. 场景定位

将“赛博小镇”的自主角色与状态世界迁移为数字工厂沙箱：设备、工序、维护、质量和能源角色在离散事件驱动下交换状态与请求，研究者可以观察瓶颈传播、告警连锁和资源竞争，并对多智能体策略进行离线分析。

沙箱用于教学与实验，不连接真实 PLC，也不声称替代专业离散事件仿真器。大模型用于角色解释和策略建议，生产节拍、资源占用和指标计算由确定性状态机完成。

## 2. 与原章内容的静态映射

| 原章知识或代码证据 | 工业扩展映射 |
| --- | --- |
| `NPCAgentManager` 与角色 Prompt | 设备、工艺、维护、质量和调度角色 |
| NPC 记忆系统 | 保存设备事件、工单和策略结果 |
| `RelationshipManager` 与好感度 | 改造成依赖强度、信任度和影响关系 |

| Godot 场景和信号 | 可视化产线、设备状态、物料流和事件传播 |

## 3. 世界模型

实体包括产线、工序、设备、缓冲区、物料批次、班组和质量门。核心状态：

- 设备：运行、待机、故障、维护、降速；
- 队列：在制品数量、等待时间和阻塞原因；
- 质量：抽检结果、缺陷类型和返工状态；
- 能源：功率、峰值和累计能耗；
- 关系：上下游依赖、共享资源、维护优先级和信息可信度。

每个状态变化由事件触发并带仿真时间、来源、前后状态和原因。

## 4. 智能体角色

- 设备智能体：解释健康状态并请求维护；
- 调度智能体：在交期、缓冲和设备可用性之间分配任务；
- 维护智能体：根据风险、备件和停机窗口排优先级；
- 质量智能体：决定抽检、隔离或追加检查建议；
- 能源智能体：提出削峰和错峰方案；
- 观察智能体：汇总 KPI 与因果事件链，不参与决策。

角色建议必须经过确定性规则校验后才改变仿真状态。

## 5. 实验设计

预置三类场景：

1. 上游设备降速导致缓冲区饥饿；
2. 质量异常触发隔离并造成交付压力；
3. 峰值电价与紧急订单形成资源冲突。

对比固定规则、单智能体和多智能体策略，记录吞吐量、在制品、停机、缺陷、能耗、工具调用和决策解释质量。

## 6. 后端与可视化

后端提供仿真初始化、单步推进、暂停、状态快照、事件流和指标接口。前端展示产线拓扑、设备颜色、物料队列、角色消息和时间线，并允许回放同一种子实验。

日志至少包含 `simulation_id`、随机种子、策略版本、事件、状态差异和决策证据，保证实验可复现。

## 8. 验收标准

- 相同种子和配置得到可复现的确定性状态轨迹；
- 大模型文本不能绕过规则直接修改世界状态；
- 支持暂停、快照、回放和事件链追踪；
- 至少比较三种策略并报告业务指标和成本；
- 明确说明沙箱模型与真实生产系统之间的适用边界。
