# 第五章 DSExt：低代码质量日报与异常闭环

> 难度：基础进阶 ｜ 对应主题：Coze、Dify、FastGPT、n8n 与可视化工作流

## 1. 场景定位

质量团队每天从 MES 导出生产与检验数据，人工汇总一次合格率、Top 缺陷和异常批次，再通过邮件或企业协作工具发布。本场景用低代码平台搭建“数据接入—指标计算—知识查询—报告生成—人工审批—消息分发”的质量日报流程。

低代码平台负责流程编排与交互，确定性指标仍由脚本或数据服务计算，避免让大模型直接心算关键质量指标。

## 3. 业务输入与指标

输入包括日批次表、检验结果表、缺陷明细表和质量标准文档。最低指标集：

- 产量、合格数、一次合格率；
- 按产线、产品和班次分组的缺陷率；
- Top-5 缺陷及环比变化；
- 越过控制限或业务阈值的异常批次；
- 数据缺失率与迟到数据数量。

质量标准、缺陷定义和处置流程进入知识库；原始测量数据不直接作为长文本塞入提示词。

## 4. 工作流设计

1. 定时触发或由用户上传指定日期的脱敏数据；
2. 校验字段、日期范围和数据量，失败则停止并通知数据责任人；
3. 调用确定性统计节点计算指标；
4. 对异常批次检索对应质量标准和历史案例；
5. 由大模型将结构化结果转为日报草稿；
6. 质量工程师审批、修改或驳回；
7. 审批通过后再发送，并保存输入版本、提示词版本和报告版本。

## 5. 平台选型练习

- Coze：适合快速构建面向人员的对话入口；
- Dify：适合组合知识库、变量和模型节点；
- FastGPT：适合突出私有知识库问答；
- n8n：适合定时任务、数据节点和外部系统编排。

学生应选择一个主平台，同时说明另外三个平台为何不作为本场景主编排器。判断维度包括数据驻留、连接器、私有化、权限、可观测性和版本迁移。

## 6. 安全与审计

- 使用脱敏样例，不上传设备密钥、客户信息和真实配方；
- 模型只生成解释，不覆盖确定性统计结果；
- 发送消息前保留人工审批节点；
- 为每次运行记录数据日期、文件摘要、工作流版本和审批人；
- 当指标计算失败或数据质量不合格时禁止生成“正常”结论。

## 8. 验收标准

- 流程含输入校验、统计、检索、生成、人工审批和审计六类节点；
- 报告中的数值与计算节点输出一致；
- 数据质量失败时流程显式终止；
- 能说明平台选择的工程权衡，而不只比较界面易用性。
