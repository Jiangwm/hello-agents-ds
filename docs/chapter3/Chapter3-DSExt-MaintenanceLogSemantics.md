# 第三章 DSExt：设备维修日志语义检索与归类

> 难度：基础 ｜ 对应主题：N-gram、分词、词向量、Transformer、提示工程与幻觉

## 1. 场景定位

工厂积累了大量非结构化维修日志，例如“主轴发热，清理风道后恢复”“伺服跟随误差，重新标定编码器”。本场景用这些短文本理解语言模型的基础机制，并构建一个小型的相似案例检索与故障类别归类实验。

场景重点是“理解模型为什么能处理文本”，不是直接搭建生产级 RAG 系统；完整知识库留到第八章。

## 2. 与原章内容的静态映射

| 原章知识或代码证据 | 工业扩展映射 |
| --- | --- |
| `N_gram.py` 的词频与条件概率 | 统计常见故障短语及设备—现象共现关系 |
| `BPE.py` 的子词合并 | 观察型号、缩写和复合故障词的切分过程 |
| `Word_Embedding.py` 的余弦相似度 | 对日志向量做相似案例排序 |
| `Transformer.py` 的注意力结构 | 分析一条日志中设备、现象、原因和措施的关联 |
| `Qwen.py` 的开源模型调用 | 用提示词抽取结构化维修事件并讨论幻觉风险 |

## 3. 数据设计

准备 100～300 条脱敏日志，每条包含：

- `log_id`、`equipment_type`、`event_time`；
- `raw_text`：原始维修描述；
- `fault_category`：机械、电气、液压、工艺或未知；
- `action_taken` 与 `downtime_minutes`；
- 可选的人工关键词和标准故障码。

训练集与验证集应按设备或时间切分，避免同一事件的近重复文本同时出现在两侧。

## 4. 分层实验

### 4.1 N-gram 基线

统计高频 unigram、bigram 和 trigram，比较“轴承 温度 高”与“电机 温度 高”等上下文概率。记录稀疏、未登录词和长距离依赖带来的问题。

### 4.2 分词与子词观察

挑选包含设备型号、缩写、单位和中英文混排的日志，比较不同分词方式。关注 `CNC-01`、`X轴`、`mm/s` 等工业符号是否被破坏。

### 4.3 向量相似检索

对日志生成向量，使用余弦相似度返回 Top-K 案例。人工检查“文字相似但原因不同”和“措辞不同但故障相同”两类误差。

### 4.4 LLM 结构化抽取

提示模型输出设备、现象、候选原因、处理措施和不确定项。所有字段必须能在原文中找到依据；原文未出现的原因应标记为推测。

## 5. 实践任务

- 基础：完成 N-gram 统计并解释三个高频共现模式。
- 核心：实现相似日志 Top-5 检索并做人工相关性标注。
- 挑战：比较词袋、静态词向量和 Transformer 表征的差异。

最终报告应包含数据切分、检索示例、错误案例和幻觉审计，不以单一准确率代替误差分析。

## 7. 验收标准

- 能解释至少一种传统语言模型与 Transformer 的能力差异；
- 相似度结果保留分数、日志 ID 和原文，便于审计；
- 结构化抽取不把模型常识混入日志事实；
- 明确记录工业缩写、单位和设备型号造成的分词问题。
