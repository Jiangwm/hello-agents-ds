# 第十六章 DSExt：预测性维护智能体综合毕业设计

> 难度：毕业设计 ｜ 对应主题：项目选题、环境、README、Notebook、测试、PR 与共创

## 1. 项目目标

完成一个端到端的预测性维护智能体：读取设备传感器与维护记录，完成数据质量检查、健康特征分析、异常或剩余寿命建模、知识检索、证据化解释和维修建议，并通过离线评测证明其有效性与安全边界。

项目重点不是追求最高模型分数，而是形成可复现的数据—模型—智能体—评测闭环。

## 2. 与原章内容的静态映射

| 原章知识或代码证据 | 毕设要求 |

## 3. 最小可交付范围

项目至少包含：

1. 一个公开或脱敏的工业时序数据集；
2. 数据字典、缺失与异常值分析；
3. 一个可解释基线模型和一个改进模型；
4. 至少两个只读工具，例如时序查询和维护知识检索；
5. 一个能根据问题选择工具并引用证据的智能体；
6. 离线测试集、指标、失败案例和安全案例；
7. 可从头运行的 Notebook 与 README。

可选高级方向包括多设备迁移、剩余寿命预测、多智能体维护计划、在线漂移监测或数字孪生沙箱联动。

## 4. 推荐数据与任务

可选公开轴承、涡扇、泵或加工设备数据，也可使用合规脱敏样例。任务从下列方向中选择一个主任务：

- 异常检测；
- 故障分类；
- 健康指数构建；
- 剩余寿命预测；
- 维修知识问答与工单推荐。

不要把多个任务简单堆叠；先让主任务形成从数据到评测的完整闭环，再增加智能体能力。

## 5. 系统架构建议

- 数据层：原始数据说明、清洗、切分和版本；
- 分析层：EDA、特征、基线、改进模型和误差分析；
- 工具层：稳定的只读数据与知识接口；
- 智能体层：任务解析、工具选择、证据聚合和回答；
- 评测层：模型指标、工具调用、报告证据性和安全；
- 展示层：Notebook 为必选，可选轻量 Web 界面。

## 6. 里程碑与人工检查点

### M1：选题与数据审查

提交问题陈述、数据许可、数据字典、风险和成功指标，确认后再建模。

### M2：基线与切分审查

完成 EDA、无泄漏切分和可解释基线；检查时间穿越、设备泄漏和类别不平衡。

### M3：智能体与工具审查

完成工具模式、权限边界和证据格式；使用 mock LLM 测试成功与失败路径。

### M4：综合评测审查

对模型、Agent、安全和成本分层评估，至少分析十个失败案例。

### M5：文档与提交审查

从干净环境按 README 复现 Notebook，检查大文件、密钥、引用、许可证和 PR 描述。

## 8. 最终验收标准

- 数据来源、许可、切分和版本清楚；
- Notebook 可从头执行，结论与生成产物一致；
- 相比基线的改进有统计或案例证据，而非只报最佳分数；
- 智能体回答可追溯到工具结果和知识来源；
- 测试覆盖成功、缺失数据、越权请求和证据不足；
- 不包含密钥、个人数据、真实生产地址或设备控制能力；
- README 能让未参与开发的人完成复现。
