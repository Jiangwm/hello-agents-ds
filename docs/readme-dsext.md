## 🎯 课程介绍

&emsp;&emsp;工业现场的数据分析往往不是一次性的算法建模：分析人员需要理解设备与工艺语义，关联时序数据、质量数据、告警、维修记录和技术文档，持续提出假设、调用工具取证、复核结论，并将结果转化为可执行且安全的业务建议。传统的数据分析课程通常侧重单一算法或 Notebook 实验，难以覆盖这种跨数据、跨系统、跨角色的长程任务。

&emsp;&emsp;本课程以 [Hello-Agents《从零开始构建智能体》](./README.md) 的 16 章技术体系为主线，将智能体基础、大语言模型、经典范式、低代码平台、多智能体框架、记忆与检索、上下文工程、通信协议、Agentic RL、性能评估和综合应用，逐章映射到“工业数据分析与挖掘”场景。

&emsp;&emsp;课程不是把工业名词简单替换进通用案例，而是围绕工业分析的真实约束重新设计任务：数据口径与单位必须明确，结论必须引用证据，相关性不能伪装成因果，数据不足时应拒答或请求补充信息，工艺调整和设备控制必须保留人工审批。学习者将从一个只会查询告警的最小智能体出发，逐步构建面向质量、设备、能耗和生产运营的多智能体分析系统。

> 本课程中的智能体用于教学、离线分析与决策支持，默认只访问脱敏样例和只读工具，不直接控制 PLC、MES、设备或生产参数。

## ✨ 你将收获什么？

- 🧠 **建立智能体技术全景**：理解 Agent、LLM、工具、记忆、上下文、协议、训练和评估之间的关系。
- 📊 **掌握工业数据分析任务建模**：将告警分诊、故障诊断、质量分析、能耗优化和预测性维护转化为可验证任务。
- 🛠️ **构建证据驱动的分析智能体**：让智能体调用确定性工具取数、计算、检索，并输出带来源的结论。
- 🤝 **设计多智能体协作流程**：划分数据、设备、工艺、质量和审核角色，建立结构化消息与人工关口。
- 🧪 **训练与评估工业智能体**：通过 SFT、GRPO、工具调用测试、综合任务、安全测试和人工评审改进系统。
- 🏭 **完成综合工业应用**：逐步形成能源分析助手、质量深度研究、数字工厂沙箱和预测性维护毕业设计。
- 🔐 **具备工业 AI 治理意识**：关注数据权限、审计、可追溯性、模型幻觉、越权操作和生产安全边界。

## 🏭 课程特色

### 1. 原理主线与工业场景双轨对应

每章保留 Hello-Agents 的原始技术目标，同时提供难度匹配的工业扩展。学习者可以先理解通用实现，再观察同一技术进入工业数据场景后，在数据语义、权限、审计和安全方面发生的变化。

### 2. 数据分析与智能体工程融合

课程同时关注数据质量、统计基线、异常检测、根因假设、文本挖掘、知识检索和预测性维护，也关注 Agent 的工具协议、状态、记忆、协作、训练、评测和应用工程。

### 3. 证据驱动，而非自由生成

关键数值由确定性分析工具计算；智能体负责理解问题、选择工具、组织证据和解释结果。所有重要结论应能回溯到数据窗口、规则、文档或工具结果。

### 4. 由浅入深的项目阶梯

课程从两个工具的单智能体开始，逐步引入多智能体、RAG、长程上下文、跨系统协议、模型训练、综合评测和全栈应用，避免一开始就陷入复杂平台搭建。

### 5. 人在回路与生产安全

调查范围、跨域数据访问、验证实验、工艺调整和最终根因结论均设置人工检查点。智能体默认只读，任何生产写操作都不属于常规教学能力。

## 💡 如何学习

### 适合人群

- 工业数据分析、数据科学、机器学习和人工智能相关专业学生；
- 制造业数据工程师、算法工程师、质量工程师和设备工程师；
- 希望从通用 Agent 开发进入工业智能应用的开发者；
- 希望把传统数据分析项目升级为可交互、可协作系统的学习者。

### 前置基础

- 掌握 Python 基础语法和常用数据结构；
- 了解 pandas、可视化和基础统计分析；
- 能理解 CSV、JSON、API 和简单的 Web 服务；
- 对大语言模型和提示词有基础认识；
- 有设备、质量、工艺或生产管理背景会更有帮助，但不是必需条件。

### 推荐学习方式

1. **先学原章**：理解该章的智能体概念、架构和通用代码。
2. **再读 DSExt**：分析工业场景、数据资产、工具边界和验收标准。
3. **建立确定性基线**：先用规则、统计或传统模型得到可复现结果。
4. **接入智能体**：让 Agent 调用基线工具，不把关键计算交给自由生成。
5. **构造失败案例**：主动测试缺失数据、单位冲突、错误参数、无证据结论和越权请求。
6. **保留实验记录**：记录数据版本、提示词、模型、工具、参数、指标和人工判断。

## 🧪 实践与考核建议

课程考核不只评价“能否运行”，还应同时检查以下维度：

| 维度 | 核心要求 |
| --- | --- |
| 数据 | 来源与许可清楚，字段和单位明确，切分无泄漏，质量问题有记录 |
| 分析 | 有确定性基线，结论与证据一致，能区分相关性、假设和因果 |
| 智能体 | 工具选择合理，参数正确，状态可恢复，失败路径清晰 |
| 工程 | 结构清楚，配置与密钥分离，关键逻辑可测试，结果可复现 |
| 评测 | 覆盖工具、任务、报告、安全、稳定性、延迟和成本 |
| 安全 | 默认只读，敏感数据脱敏，越权请求被拒绝，关键决策有人审 |

建议采用“章节实践 + 阶段项目 + 综合毕业设计”的方式：

- 第一阶段提交规则或单智能体分析报告；
- 第二阶段提交带工具注册和测试的工业 Agent；
- 第三阶段提交 RAG、长程调查或评测项目；
- 第四阶段完成一个综合应用；
- 第五阶段完成可复现的预测性维护或自选工业毕业设计。

## 🔐 数据、模型与安全原则

- 不提交 API Key、令牌、真实生产地址、个人数据、客户数据或未脱敏工艺数据；
- 公开数据必须保留来源、许可和版本，私有数据只提供结构兼容的脱敏样例；
- 原始数据与派生数据分离，处理过程保留参数和数据摘要；
- 常规测试使用固定样例和 mock 响应，不依赖付费 API；
- 模型输出必须标记事实、假设、未知项和建议，禁止伪造测量值或来源；
- 所有生产控制、参数写入和高风险建议必须脱离教学环境并经过正式审批。
