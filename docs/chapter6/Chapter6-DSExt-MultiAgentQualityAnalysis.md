# 第六章 DSExt：多智能体质量根因分析团队

> 难度：进阶 ｜ 对应主题：AutoGen、AgentScope、CAMEL、LangGraph 与框架选型

## 1. 场景定位

针对复杂质量异常，单个智能体容易在统计、设备、工艺和质量知识之间失焦。本场景组建一个多智能体分析团队，对某产品尺寸漂移问题进行协同取证、假设竞争和报告审核。

目标不是让智能体“开会越多越好”，而是将职责、消息格式、终止条件和人工审批设计成可验证的协作流程。

## 2. 与原章内容的静态映射

| 原章知识或代码证据 | 工业扩展映射 |
| --- | --- |
| AutoGen 软件团队的产品、工程、评审角色 | 数据分析师、工艺工程师和质量审核员 |
| AgentScope 的结构化输出与投票 | 对候选根因进行证据评分和受控表决 |
| CAMEL 的角色扮演协作 | 让设备专家与质量专家围绕一个明确任务交换证据 |
| LangGraph 的 `SearchState` 与节点路由 | 构建确定的取数—分析—复核状态图 |

## 3. 角色与职责

- **协调智能体**：解析问题、维护任务状态，不直接给根因结论；
- **数据分析智能体**：构造对照组、计算漂移和相关性；
- **工艺智能体**：将异常参数映射到工艺机理与规范；
- **设备智能体**：检查告警、维护和部件状态；
- **质量审核智能体**：检查证据强度、混杂因素和报告措辞；
- **人工负责人**：批准验证试验与最终结论。

每条跨智能体消息至少包含 `task_id`、`evidence_ids`、`claim`、`confidence`、`open_questions`，避免只传递自由文本观点。

## 4. 协作流程

1. 协调智能体把问题拆分为数据、工艺、设备三条调查线；
2. 三个专业智能体并行形成带证据 ID 的候选假设；
3. 协调智能体合并重复假设，并要求补充相互冲突的证据；
4. 审核智能体按证据完整性、时间一致性和可反证性评分；
5. 若最高分仍低于阈值，返回补充取数节点；
6. 达到阈值后生成报告草稿，交人工负责人确认。

终止条件应限制最大轮次、最大工具调用数和最低新增证据量，防止无效对话循环。

## 5. 框架对比任务

- 用 AutoGen 表达角色对话与审核链；
- 用 AgentScope 表达结构化消息、并行决策和投票；
- 用 CAMEL 表达两个专业角色的深度协作；
- 用 LangGraph 表达可回放、可中断的状态机。

最终选择一个主实现，并从状态可见性、并发、恢复、类型约束、人工介入和调试成本解释选择。

## 6. 数据与输出

输入包括批次质量、设备告警、工艺时序、维护记录和工艺规范。输出为：

- 候选根因清单及证据图；
- 被排除假设及反证；
- 缺失数据和补充采集请求；
- 由人工批准的验证实验计划；
- 完整消息与工具调用轨迹。

## 8. 验收标准

- 每个角色职责互斥且有明确输入输出；
- 候选根因必须引用证据 ID，投票不能代替证据；
- 协作流程支持证据不足回退和人工中断；
- 能用同一案例比较至少两种框架的工程差异。
