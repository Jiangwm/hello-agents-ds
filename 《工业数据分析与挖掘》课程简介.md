## 二、课程定位

本课程面向工业互联网、智能制造、工业大数据及人工智能相关专业，是连接“数据分析与机器学习基础”与“工业智能系统开发及综合实践”的专业核心课程。课程以工业现场真实工作过程为主线，以智能体（Agent）技术体系为创新载体，组织工业多源数据理解、异常识别、知识检索、根因分析、预测决策、系统协作和质量评测等学习任务，培养学生将数据、模型、工具、知识与业务流程集成为可复现、可审计、可协作的工业数据分析系统的能力。

课程面向工业数据分析工程师、工业 AI 应用开发工程师、智能制造数据工程师、质量数据分析工程师、设备健康与预测性维护工程师等岗位（群），重点对接以下典型工作任务：

1. 获取、清洗、关联和解释设备、工艺、质量、能耗、告警、工单及技术文档等多源异构数据；
2. 构建统计、规则、机器学习与大语言模型相结合的确定性分析基线；
3. 设计可调用数据工具、知识库和专业模型的工业数据分析智能体；
4. 完成生产异常、质量波动、设备故障、能源优化等问题的证据化调查与报告；
5. 建立多智能体协作、长程上下文、工业协议连接、性能评测及安全治理机制；
6. 面向真实业务约束完成工业智能应用的设计、开发、测试、部署说明和持续改进。

课程为后续综合实训、智能制造系统集成及毕业设计奠定工业数据建模、智能体工程和项目实施基础，同时强化数据合规、生产安全、绿色制造和持续学习意识，支持学生面向产业数字化升级形成可持续发展能力。

## 四、课程目标

完成本课程后，学生应达到以下目标：

1. **专业知识目标**：理解工业数据的时序性、多源异构性、语义依赖和质量约束，掌握智能体、LLM、工具调用、经典 Agent 范式、记忆与 RAG、上下文工程、多智能体协议、Agentic RL 和性能评估等核心知识。
2. **数据分析目标**：能够根据设备、质量、工艺、能源和生产运营问题建立数据字典与分析口径，完成数据质量检查、统计基线、规则诊断、语义检索、异常分析和预测性维护等任务，区分事实、相关性、假设与因果结论。
3. **智能体开发目标**：能够将工业分析任务分解为“感知—推理—工具—行动—复核”闭环，设计只读工具、结构化输入输出、记忆和协作机制，开发可测试、可恢复、可追溯的工业数据分析智能体。
4. **工程创新目标**：能够综合运用低代码平台、Agent 框架、MCP/A2A/ANP 协议和模型优化方法，完成从需求分析、基线构建、系统实现到分层评测的端到端项目，并围绕工业场景提出具有可实施性的创新方案。
5. **职业能力目标**：能够以工程文档、数据证据、代码与测试、可视化报告和项目答辩表达成果，具备跨数据、设备、工艺、质量和管理角色协作的能力。
6. **职业素养与课程思政目标**：形成质量第一、精益求真、工匠精神、协作担当和自主创新意识；遵守数据安全、知识产权、算法伦理与生产安全规范，坚持关键结论证据可追溯、高风险操作人工审批、工业智能体默认只读的责任边界。

## 五、课程特色

1. **原理与工业场景双轨融合**：以 16 章智能体技术主线为骨架，同步嵌入设备告警、维修日志、质量根因、良率调查、能耗优化和预测性维护等工业任务，避免将通用案例简单改名为工业案例。
2. **突出职业本科的系统性和实践性**：相较于偏重工具操作的专科课程，本课程强化原理理解、框架抽象、协议连接、训练评测与系统集成；相较于偏重算法推导的应用本科同类课程，本课程更加突出岗位任务、工程流程、项目交付和生产约束。
3. **构建“数据—模型—工具—智能体—评测”创新能力链**：不止训练单一算法或完成一次 Notebook 分析，而是让学生把确定性算法、专业知识、数据服务和 Agent 工作流组合为可持续迭代的工业智能应用。
4. **坚持证据驱动与人在回路**：关键数值由规则、统计或模型工具计算；智能体负责问题理解、工具选择、证据组织和结果解释。数据不足时应请求补充信息或拒绝下结论，工艺调整、设备控制和其他高风险建议必须经过人工审批。
5. **项目阶梯贯穿全程**：从最小告警分诊智能体起步，逐步发展到多智能体协作、RAG、长程调查、跨系统协议、训练评测、数字工厂沙箱和预测性维护综合项目。
6. **融入产业报国与工程伦理**：将数据主权、工业安全、绿色制造、质量强国、工匠精神、团队协作和关键技术自主创新融入案例约束、项目验收与评价标准，形成课程特色思政资源。

课程可向学校“双高”4+N 资源体系贡献“工业数据真实性与质量责任”“智能体越权操作与生产安全”“关键工业软件自主创新”“绿色制造与能源优化”“多岗位协作与工匠精神”等案例资源。具体资源分类、编号和归属应按学校当期“双高”建设任务书认定。

## 六、课程内容概要（项目化思路）

课程采用 12 个递进式项目化学习单元，共 48 学时。每个单元以典型工业工作任务组织“知识学习—基线实现—智能体扩展—失败案例测试—复盘改进”。

| 单元 | 学时 | 典型工作任务或知识模块 | 主要教学内容与要求 | 教学方式 | 对未来工作或后继学习的作用 |
| --- | ---: | --- | --- | --- | --- |
| 1. 设备告警分诊智能体 | 4 | 设备告警理解与分级处置 | 理解 Agent 的定义、构成和行动循环；建立传感器摘要、告警规则、工具调用与处置建议的最小闭环；明确只读访问和人工审批边界 | 知识讲解、案例拆解、最小项目实践 | 支撑设备监测、告警管理和现场异常初判，建立智能体系统整体认知 |
| 2. 规则驱动故障问诊 | 4 | 泵组故障问诊与规则知识库 | 理解符号主义、专家系统及学习范式演进；设计故障规则、追问策略、置信度和未知状态，分析规则覆盖范围与失效边界 | 规则实验、情境问诊、结对复核 | 支撑设备故障初诊和可解释决策，形成“先基线、后智能”的工程意识 |

## 七、课程设计概要

课程内容遵循“基础认知—系统构建—高级能力—综合应用—创新交付”的递进逻辑：

```mermaid
flowchart LR
    A["工业问题与数据理解"] --> B["规则/统计/机器学习基线"]
    B --> C["单智能体与工具调用"]
    C --> D["多智能体、记忆与上下文"]
    D --> E["协议连接、训练与评测"]
    E --> F["工业综合应用"]
    F --> G["可复现、可审计的创新项目"]
```

教学实施坚持以下原则：

- **任务驱动**：每个单元从岗位问题和验收标准出发，再引入必要知识和技术；
- **理实一体**：理论讲授约占 33.3%，项目实践约占 66.7%，课堂同步完成数据分析、工具开发和复盘；
- **确定性基线优先**：先以规则、统计或传统模型获得可复现结果，再由智能体编排工具与解释证据；
- **逐级集成**：单工具、单智能体、多智能体、跨系统协作、综合项目逐步增加复杂度；
- **全过程治理**：统一记录数据版本、提示词、模型、工具、参数、指标和人工判断，测试缺失数据、单位冲突、错误参数、无证据结论及越权请求；
- **教学安全边界**：仅使用公开、脱敏或合成数据及只读接口，不连接真实生产控制，不允许智能体直接写入 PLC、MES 或设备参数。

## 八、评价方式

课程采用过程性评价与结果性评价相结合的方式，过程性评价占 60%，结果性评价占 40%。评价不仅检查“能否运行”，还覆盖数据、分析、智能体、工程、评测与安全六个维度。

| 评价类型 | 考核环节 | 比例 | 目标分值 | 考核细则 |
| --- | --- | ---: | ---: | --- |
| 过程性评价 | 12 个项目化单元 | 50% | 100 | 检查各单元项目的任务完成度、数据口径、确定性基线、工具调用正确性、证据引用、工程实现及失败案例处理 |
| 过程性评价 | 工程过程与学习档案 | 10% | 100 | 检查数据版本、实验记录、代码结构、配置与密钥分离、测试、README、问题清单和改进记录 |
| 结果性评价 | 期末考试 | 40% | 100 | 综合考查工业数据分析与挖掘、智能体技术原理、典型工业场景分析、系统设计、性能评测及安全治理等内容 |
|  | **合计** | **100%** |  |  |

### 评价维度与底线要求

| 维度 | 核心要求 |
| --- | --- |
| 数据 | 来源与许可清楚，字段、单位和时间窗口明确，数据切分无泄漏，质量问题有记录 |
| 分析 | 有可复现基线，结论与证据一致，明确区分事实、相关性、假设和因果 |
| 智能体 | 工具选择合理、参数正确、状态可恢复、错误可解释、失败路径清晰 |
| 工程 | 项目结构清楚，配置与密钥分离，关键逻辑有测试，依赖和运行步骤完整 |
| 评测 | 覆盖工具调用、任务结果、报告质量、安全性、稳定性、延迟和成本 |
| 安全 | 默认只读，敏感数据脱敏，越权请求被拒绝，高风险建议设置人工审批 |

出现伪造数据或来源、泄露密钥或敏感数据、绕过人工审批执行高风险操作等情形，相关考核环节不得评定为合格。

## 九、预期课程成果

学生应形成可进入个人作品集或后续综合实训的成果包，包括：

1. 工业数据字典、分析口径和数据质量记录；
2. 至少一个规则、统计或机器学习确定性分析基线；
3. 一个可调用只读工具并输出证据链的工业数据分析智能体；
4. 一个多智能体、RAG、长程调查或协议协作阶段项目；
5. 一套覆盖正常路径、失败路径和安全边界的评测数据与测试记录；
6. 一个具有 README、运行说明、结果报告和答辩材料的期末综合项目。
